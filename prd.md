# PostureGuard PRD

## 1. Product Overview

PostureGuard는 노트북 및 데스크탑 브라우저에서 동작하는 웹캠 기반 AI 건강 코치 웹앱이다. 사용자의 카메라 영상을 외부 서버로 전송하지 않고 브라우저 내부에서 MediaPipe Face Mesh로 분석해 자세, 눈 피로, 휴식 루틴을 조용히 안내한다.

제품의 핵심 방향은 “자세를 감시하는 앱”이 아니라 “업무 중 사용자를 조용히 챙겨주는 AI 웰니스 코치”이다.

## 2. Goals

- 사용자가 별도 설치 없이 URL 접속만으로 자세 모니터링을 시작할 수 있게 한다.
- 웹캠 영상은 On-Device로만 처리해 개인정보 부담을 줄인다.
- 업무 중 방해되지 않도록 monitoring 모드에서는 작은 건강 코치 Orb만 표시한다.
- 나쁜 자세, 눈 휴식, 스트레칭을 부드러운 코칭 메시지로 안내한다.
- 데스크탑 브라우저에서 실제 배포 가능한 SaaS 품질의 UI/UX를 제공한다.

## 3. Non-Goals

- 모바일 브라우저 대응은 현재 범위에 포함하지 않는다.
- 서버 저장, 계정, 로그인, 클라우드 동기화는 제공하지 않는다.
- 의료 진단 또는 치료 목적의 기능은 제공하지 않는다.
- 카메라 영상 업로드, 녹화, 서버 분석은 하지 않는다.

## 4. Target Users

- 장시간 노트북/데스크탑으로 일하는 사무직 사용자
- 화상회의, 문서 작업, 개발, 디자인 등 장시간 앉아서 일하는 사용자
- 자세 교정 앱의 강한 경고보다 조용한 건강 코칭을 원하는 사용자

## 5. Supported Environment

- Windows Chrome
- Windows Edge
- Mac Chrome
- Mac Safari
- HTTPS 배포 환경
- localhost 개발 환경

카메라 권한 정책상 배포 환경은 HTTPS가 필요하다.

## 6. Core User Flow

1. 사용자가 PostureGuard URL에 접속한다.
2. 최초 사용자 가이드를 확인하고 시작한다.
3. 브라우저 카메라 권한을 허용한다.
4. setup 화면에서 카메라를 보며 기준 자세를 저장한다.
5. 모니터링 시작 버튼을 누른다.
6. 앱은 monitoring 모드로 전환되고 작은 건강 코치 Orb만 표시된다.
7. 자세가 흐트러지면 부드러운 토스트 메시지와 선택적 비프음으로 안내한다.
8. 사용자가 Orb를 클릭하면 preview 화면에서 통계와 설정을 확인할 수 있다.

## 7. Functional Requirements

### 7.1 Camera and MediaPipe

- MediaPipe Face Mesh를 CDN으로 로드한다.
- 웹캠 영상은 브라우저 내부에서만 처리한다.
- 카메라 영상은 setup/preview 모드에서 표시하고 monitoring 모드에서는 숨긴다.
- 카메라 권한 거부, 카메라 없음, 카메라 사용 중, HTTPS 필요, 브라우저 미지원 상황을 안내한다.

### 7.2 Calibration

- 사용자가 기준 자세 저장 버튼을 누르면 현재 얼굴 랜드마크를 기준값으로 저장한다.
- 저장 기준값:
  - 눈 사이 거리
  - 코 Y 좌표
  - 턱 Y 좌표
  - 양쪽 눈 Y 차이
  - noseOffsetX
  - 얼굴 중심 X/Y
- 얼굴이 화면 중앙에서 크게 벗어난 경우 기준 자세 저장을 막고 안내한다.

### 7.3 Posture Detection

감지 상태:

- 좋은 자세
- 화면 가까움
- 고개 앞으로 내려감
- 눈 긴장
- 고개 좌우 기울어짐
- 얼굴 좌우 회전
- 얼굴 위치 치우침
- 얼굴 감지 안 됨

판별 우선순위:

1. 얼굴 감지 안 됨
2. 얼굴 위치 치우침
3. 얼굴 좌우 회전
4. 고개 좌우 기울어짐
5. 화면 가까움
6. 고개 앞으로 내려감
7. 눈 긴장
8. 정상

### 7.4 Health Coach Orb

- monitoring 모드에서는 우측 중앙에 작은 캡슐형 Orb만 표시한다.
- Orb 내부에는 건강 코치 얼굴 이모지를 표시한다.
- Orb 클릭 시 preview 모드로 전환한다.
- Orb 표정은 나쁜 자세 누적 시간에 따라 변한다.

표정 기준:

- 0~10분: 😊
- 10~30분: 🙂
- 30~60분: 😐
- 60분 이상: 😣

좋은 자세가 유지되면 나쁜 자세 누적 시간이 천천히 감소한다.

### 7.5 Toast Notification

- 업무를 방해하지 않는 작은 토스트를 표시한다.
- 앱 이름을 토스트에 표시하지 않는다.
- 같은 경고는 10초 쿨다운을 적용한다.
- 경고음 끄기 옵션이 켜져 있으면 Web Audio 비프음을 재생하지 않는다.

### 7.6 Health Statistics

오늘의 통계:

- 자세 점수
- 좋은 자세 유지 시간
- 자세 교정 횟수
- 상태별 발생 횟수
- 스트레칭 완료 횟수
- 다음 눈 휴식까지 남은 시간
- 다음 스트레칭까지 남은 시간

자세 점수:

- 초기값 100점
- 나쁜 자세 이벤트 발생 시 1점 감점
- 같은 이벤트는 30초에 한 번만 감점

### 7.7 Eye Rest

- 20분마다 눈 휴식 토스트를 표시한다.
- 메시지: “20초 동안 먼 곳을 바라볼까요?”
- 휴식 완료 버튼을 누르면 20분 타이머를 다시 시작한다.
- preview 모드에서 20초 테스트 버튼을 제공한다.

### 7.8 Stretch Reminder

- 1시간마다 앉아서 할 수 있는 스트레칭 안내를 표시한다.
- 사용자는 스트레칭 하기, 나중에 하기, 오늘은 끄기를 선택할 수 있다.
- 나중에 하기는 15분 뒤 다시 안내한다.
- 오늘은 끄기는 해당 날짜 동안 localStorage로 저장한다.
- 스트레칭 완료 시 완료 횟수를 1 증가시키고 다음 1시간 타이머를 시작한다.
- preview 모드에서 스트레칭 바로 보기와 30초 테스트 버튼을 제공한다.

스트레칭 동작:

- 목 좌우 천천히 기울이기
- 어깨 돌리기
- 손목 스트레칭
- 등 펴고 깊게 숨쉬기

### 7.9 Settings

설정 메뉴 항목:

- 오늘 통계 초기화
- 기준 자세 다시 설정
- 모든 데이터 초기화
- About
- 데스크탑 설치

데이터 초기화 작업에는 확인창을 표시한다.

### 7.10 Onboarding

- 최초 실행 시 사용 방법을 1회 표시한다.
- 시작하기를 누르면 localStorage에 저장해 다시 표시하지 않는다.

### 7.11 About

About 화면에는 다음 내용을 표시한다.

- 제품명
- 제품 설명
- 기술 스택
- 개인정보 보호 안내

## 8. Data Storage

데이터는 브라우저 LocalStorage에 저장한다.

저장 항목:

- 자세 점수
- 좋은 자세 유지 시간
- 나쁜 자세 누적 시간
- 상태별 통계
- 마지막 리셋 날짜
- 눈 휴식 타이머
- 스트레칭 비활성화 날짜
- 스트레칭 완료 횟수
- 다음 스트레칭 예정 시간
- 온보딩 확인 여부

날짜가 바뀌면 일일 통계는 자동 초기화한다.

## 9. Privacy Requirements

- 카메라 영상은 서버로 전송하지 않는다.
- 카메라 영상은 녹화하지 않는다.
- 분석은 브라우저 메모리 안에서만 수행한다.
- 통계는 사용자 브라우저 LocalStorage에만 저장한다.

## 10. PWA and Deployment

- Vite 기반 정적 사이트로 배포한다.
- manifest.webmanifest를 제공한다.
- service worker로 기본 앱 shell을 캐시한다.
- 데스크탑 설치형 웹앱을 지원한다.
- 권장 배포 플랫폼은 Vercel이다.

## 11. Success Metrics

- 사용자가 카메라 권한 허용 후 1분 내 기준 자세 저장을 완료한다.
- monitoring 모드에서 업무 화면 방해가 최소화된다.
- 사용자가 Orb를 통해 상태를 빠르게 이해한다.
- 토스트 메시지가 경고보다 코칭으로 인식된다.
- 하루 통계와 휴식 안내가 재방문 동기를 제공한다.

- Orb 클릭 확인

- 눈 휴식 알림 확인

- 설정 메뉴 초기화 기능 확인

- 오프라인 앱 shell 확인
- README 및 배포 체크리스트 확인
